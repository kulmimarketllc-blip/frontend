System Architecture

ESUUQ follows a layered monolith architecture with NestJS modules, ready to extract into microservices. All external requests go through API Gateway → NestJS app → service layer → database.

Client (Web/Mobile)
→
CloudFront CDN
→
AWS ALB
→
NestJS API
→
PostgreSQL + Redis
🏗️ NestJS Modules
  • AuthModule — JWT, OAuth, OTP
  • UsersModule — customers, profiles
  • MerchantsModule — stores, onboarding
  • ProductsModule — listings, search
  • OrdersModule — lifecycle, tracking
  • PaymentsModule — Stripe integration
  • DeliveryModule — drivers, routing
  • NotificationsModule — email, SMS, push
  • AdminModule — platform management
  • AnalyticsModule — reporting, metrics
☁️ AWS Infrastructure
  • EC2 / ECS (Fargate) — API containers
  • RDS PostgreSQL (Multi-AZ)
  • ElastiCache Redis (cluster mode)
  • S3 + CloudFront — static assets, images
  • ALB — load balancing, health checks
  • Route 53 — DNS management
  • ACM — SSL/TLS certificates
  • CloudWatch — logs, metrics, alarms
  • SES — transactional email backup
  • Secrets Manager — env vars, keys
🔒 Security Layers
  • JWT access (15min) + refresh (7d) tokens
  • bcrypt password hashing (rounds: 12)
  • Rate limiting via Redis (100 req/15min)
  • Helmet.js HTTP security headers
  • CORS restricted to whitelisted origins
  • Input validation via class-validator
  • SQL injection protection (TypeORM)
  • Role-based access control (RBAC)
  • IP blocking on repeated failures
  • Optional 2FA (TOTP / SMS)
📡 Real-time (WebSockets)
  • Socket.IO via NestJS Gateway
  • driver:location — GPS position updates
  • order:status — live status changes
  • new:order — merchant order alerts
  • chat:message — customer ↔ driver chat
  • admin:metrics — live dashboard data
  • Rooms by order ID for scoped events

Authentication Flow

All protected routes require a Bearer JWT token in the Authorization header. Tokens expire in 15 minutes; use the refresh endpoint to obtain new ones.

Register / Login
→
OTP Verify (email + SMS)
→
Issue JWT + Refresh Token
Access Token Expires
→
POST /auth/refresh
→
New Access Token
Logout
→
Blacklist Refresh Token in Redis

Error Handling

All errors return a consistent JSON envelope. HTTP status codes follow REST conventions.

// Standard error response { "statusCode": 400, "error": "Bad Request", "message": "email must be a valid email address", "timestamp": "2026-03-12T14:22:00Z", "path": "/v1/auth/register" }
200 OK 201 Created 204 No Content 400 Bad Request 401 Unauthorized 403 Forbidden 404 Not Found 409 Conflict 422 Unprocessable 429 Rate Limited 500 Server Error

Response 200
{ "accessToken": "eyJhbGci...", // expires 15min "refreshToken": "eyJhbGci...", // expires 7 days, stored httpOnly cookie "user": { "id": "usr_01HZ4K...", "firstName": "Ahmed", "email": "[email protected]", "role": "customer", "verified": true } }

Product Endpoints

GET /products List products with filters & pagination Public
Query Parameters
?page=1&limit=20 // pagination &category=electronics // filter by category slug &merchant=merch_01HZ... // filter by merchant ID &q=wireless+earbuds // full-text search (pg_trgm) &minPrice=10&maxPrice=100 // price range &inStock=true // only in-stock items &sort=price_asc // price_asc | price_desc | newest | popular &featured=true // featured products only
Response 200
{ "data": [ { "id": "prod_01HZ...", "name": "Wireless Earbuds Pro Max", "slug": "wireless-earbuds-pro-max", "price": 49.99, "comparePrice":89.99, "stock": 42, "images": ["https://cdn.esuuq.com/..."], "merchant": { "id": "merch_...", "name": "TechZone MN" }, "rating": 4.9, "reviewCount": 834, "category": "electronics" } ], "meta": { "total": 10482, "page": 1, "limit": 20, "pages": 524 } }
POST /products Create new product listing merchant
Request Body (multipart/form-data)
{ "name": "Wireless Earbuds Pro Max", "description": "Premium sound quality...", "price": 49.99, "comparePrice": 89.99, // optional "stock": 100, "sku": "TZ-EAR-001", "categoryId": "cat_electronics", "lowStockAt": 10, // alert threshold "variants": [{ "type": "color", "values": ["Black", "White"] }], "images": // File[] — uploaded to S3, CDN URL returned }

Notes

Partial update supported (PATCH semantics despite PUT). Only the owning merchant or an admin can update. Stock updates trigger low-stock check and notifications.

Payment Endpoints

POST /payments/intent Create Stripe PaymentIntent customer
Request Body
{ "amount": 9998, // in cents "currency": "usd", "method": "card" // card | paypal | apple_pay | google_pay }
Response 201
{ "clientSecret": "pi_3OxK2...secret_...", // passed to Stripe.js on frontend "intentId": "pi_3OxK2..." }
POST /payouts/request Merchant / driver payout request merchant | delivery_partner
Flow

Balance check → create Stripe Transfer to connected account → update payout record → send confirmation email. Minimum payout: $20. Processing time: 1–2 business days.
